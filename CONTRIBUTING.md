# Como contribuir al Agent Canvas Model

Gracias por tu interes en mejorar el Agent Canvas. Este es un proyecto abierto y las contribuciones de la comunidad son fundamentales para que el framework evolucione.

## Formas de contribuir

### 1. Compartir ejemplos
Si has usado el Agent Canvas para disenar un agente, comparte tu experiencia:
- Crea un archivo en `ejemplos/` siguiendo el formato de los ejemplos existentes
- No incluyas informacion confidencial de tu organizacion
- Describe que funciono, que no, y que aprendiste

### 2. Mejorar las plantillas
- Proponer cambios a los bloques existentes
- Anadir nuevos formatos (Figma, Miro, Notion...)
- Mejorar el diseno visual de las plantillas HTML

### 3. Traducir
- Traducir el framework a otros idiomas
- Mantener la coherencia terminologica con el original en espanol

### 4. Documentacion
- Ampliar la guia de mejores practicas
- Anadir guias especificas para plataformas (AWS, GCP, Azure, etc.)
- Mejorar los ejemplos existentes

## Como enviar tu contribucion

1. Haz fork del repositorio
2. Crea una rama con nombre descriptivo: `feature/ejemplo-agente-ventas` o `fix/typo-readme`
3. Realiza tus cambios
4. Envia un Pull Request con una descripcion clara de lo que cambiaste y por que

## Principios

- **Claridad sobre exhaustividad**: mejor un bloque bien explicado que diez mal definidos
- **Agnostico de plataforma**: el framework debe funcionar independientemente del stack tecnologico
- **Orientado a la practica**: prioriza lo que ayuda a equipos reales a disenar mejores agentes
- **Accesible**: el lenguaje debe ser comprensible para perfiles tecnicos y de negocio

## Reportar problemas

Usa la seccion de Issues de GitHub para:
- Reportar errores en las plantillas
- Sugerir mejoras o nuevos bloques
- Proponer cambios en la metodologia

---

Gracias por contribuir. Cada mejora ayuda a que mas equipos diseñen mejores agentes IA.
