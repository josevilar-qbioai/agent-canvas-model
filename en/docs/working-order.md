# Suggested Working Order for the Agent Canvas

The Agent Canvas has 9 blocks. Don't try to fill them all in a single session. The suggested order reflects where ambiguity is most costly if left unresolved.

---

## Phase 1: Define the Problem (Blocks 1–2)

### Block 1 — User Segment
**Key question**: Who is this agent for?

Before defining what the agent does, define who it does it for. Identify:
- Primary users (use the agent daily)
- Secondary users (use it occasionally)
- Specific needs of each segment

**Advancement criterion**: you can describe your primary user in one concrete, non-generic sentence.

### Block 2 — Value Proposition
**Key question**: What problem does it solve?

This is the most important block. If it's not clear, do not move forward.
- What specific problem does it solve?
- What benefits does the user get? (time savings, error reduction, better experience)
- Why is an AI agent a better solution than current alternatives?

**Advancement criterion**: you can explain the value in 30 seconds to someone who doesn't know the project.

---

## Phase 2: Define the Experience (Blocks 3–5)

### Block 3 — Channels
**Key question**: How do users interact with the agent?

Don't multiply channels unnecessarily. Start with ONE — the channel where your users already are.

### Block 4 — Relationship / Personality
**Key question**: What role and tone does the agent have?

Define: is it an assistant, an expert, a tutor? Formal or conversational? This directly affects the system prompt and user experience.

### Block 5 — KPIs (Value Sources)
**Key question**: How do we measure whether it delivers value?

Define metrics BEFORE developing:
- Time saved (hours/week)
- First-interaction resolution rate
- User satisfaction (NPS)
- Reduction in support tickets

---

## Phase 3: Define the Solution (Blocks 6–8)

### Block 6 — Key Resources
What knowledge, data, or APIs does it need?

### Block 7 — Key Activities (Skills)
What capabilities does it have? Start with 1–2 well-built skills.

### Block 8 — Partners / Integrations
What systems does it connect to? Each integration adds complexity — prioritize what's essential.

---

## Phase 4: Assess Feasibility (Block 9)

### Block 9 — Costs & Risks
Development costs, licenses, maintenance. Technical, business, and regulatory risks.

---

## Process Rules

1. **Don't fill everything in one session** — it's a living document
2. **Validate each block with real users**
3. **Review every 2–3 weeks** during development
4. **If block 2 is not clear, stop** — do not move forward
